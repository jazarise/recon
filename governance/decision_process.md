# Decision Process

To prevent ReconX from bloating into a chaotic collection of scripts (as it was in v1.0), all architectural decisions follow this process:

1. **RFC Proposal:** A contributor submits an Issue labeled `RFC` (Request for Comment) detailing the proposed change.
2. **Community Discussion:** Minimum 7-day discussion period.
3. **Maintainer Review:** Core Maintainers evaluate the RFC against the `architecture_principles.md`.
4. **Vote:** Simple majority of Core Maintainers required to approve.
5. **Implementation:** If approved, a PR can be drafted. PRs for major architectural shifts without an approved RFC will be closed.
