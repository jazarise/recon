Environment Setup
-----------------

From PowerShell:

1. Open the ReconX workspace.

   cd E:\ReconX

2. Create a virtual environment.

   python -m venv .venv

3. Activate the virtual environment.

   .\.venv\Scripts\Activate.ps1

4. Upgrade pip.

   python -m pip install --upgrade pip

5. Install dependencies.

   python -m pip install -r requirements.txt


If Python is not on PATH
------------------------

In this Codex desktop environment, a bundled Python runtime is available at:

C:\Users\jaish\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe

You can use it directly for tests and compile checks:

C:\Users\jaish\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe -m unittest discover .\testing\unit

For normal local development, installing Python and using a project virtual
environment is cleaner.

Basic Usage Flow
----------------

1. Onboard a tenant.

   POST /api/v1/ecosystem/tenants/onboard

   Example body:

   {
     "organization_name": "Acme Security",
     "admin_email": "admin@example.com",
     "admin_name": "Admin User",
     "initial_project": "External Attack Surface"
   }

   This creates:
   - tenant
   - owner user
   - Security Operations team
   - initial project
   - owner RBAC grant
   - audit ledger entries

2. Use the returned tenant_id, user_id, team_id, and project_id for later calls.

3. Create or list projects and teams.

4. Add findings to the knowledge base.

5. Create SOC cases from findings.

6. Add comments, evidence, assignments, workflow history, and remediation status
   to cases.

7. Create recurring schedules for workflows.

8. Use global search to find findings, notes, cases, telemetry, workflows, and
   historical memory.

9. Generate executive reports and dashboard snapshots.

10. Export audit/compliance reports for accountability.


Security and Governance Model
-----------------------------

Every Stage 14 operation should remain:
- tenant-scoped
- RBAC-gated
- audit-logged
- explainable
- modular
- sandbox-aware

Tenant isolation is handled by:
E:\ReconX\workspaces\tenants\isolation.py

RBAC is handled by:
E:\ReconX\workspaces\permissions\rbac.py

Audit records are hash-linked by:
E:\ReconX\lifecycle\auditing\ledger.py

Cross-tenant access should be denied. Team-scoped roles inherit only to projects
that belong to that team.
